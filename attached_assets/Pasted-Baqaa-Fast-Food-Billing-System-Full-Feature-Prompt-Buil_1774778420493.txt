Baqaa Fast-Food Billing System — Full Feature Prompt
Build a Point of Sale (POS) billing system for a fast-food restaurant called “Baqaa Fast Food” using React + TypeScript + Tailwind CSS + shadcn/ui. All data is stored in localStorage (no backend needed). The app uses HashRouter for routing.

Pages & Routes
/ — Cashier Billing Screen (Main POS)
/admin — Admin Panel (Menu & Shop Management)
/analytics — Analytics Dashboard (Order History & Reports)
Page 1: Cashier Billing Screen (/)
Layout: Full-screen, split into two sections — Menu (left, 2/3 width on desktop) and Cart (right, 1/3 width). On mobile, they stack vertically.

Menu Section:

Category filter dropdown (default: “All Categories”)
Grid of menu item cards (2 cols mobile, 3 cols desktop) showing: emoji icon, item name, price in ₹, and “Add” button
Clicking a card or the Add button adds the item to the cart
Cart Section (right sidebar):

Customer Name field (required) — with auto-suggest from previously saved customers
Phone Number field (optional, tel input) — auto-fills when selecting a returning customer
Cart Items list — each item shows name, unit price, quantity controls (+/- buttons), remove button, and line total
Discount Section:
Dropdown: “No Discount”, “Percentage (%)”, “Rupees (₹)”
When Percentage or Rupees is selected, an input field appears to enter the discount value
Percentage discount caps at 100%, Rupees discount caps at subtotal
Totals Display: Subtotal, Discount amount (in red, showing type), and bold Total
Payment Method dropdown: Cash or Online
Generate Bill button (disabled if cart is empty or no customer name)
After Generate Bill — Receipt Screen:

Full-screen receipt view with shop logo, name, address, contact
Bill number, date/time, customer name, phone, payment method
Items table: Item, Qty, Price, Amount
Subtotal, Discount (if any), and Total
“Thank You! Visit Again!” footer
Three action buttons at the top (hidden on print):
🖨️ Print — triggers window.print()
📱 WhatsApp — opens WhatsApp with a formatted text bill (auto-fills customer phone with +91 prefix)
✕ Close — returns to billing screen and resets cart
Page 2: Admin Panel (/admin)
Shop Info Management: Edit shop name, address, contact, GSTIN, logo
Category Management: Add/delete categories
Menu Item Management: Add/edit/delete items with name, price, category, and image
Navigation buttons to go back to POS or Analytics
Page 3: Analytics Dashboard (/analytics)
Time Filter: Daily / Weekly / Monthly
Stats Cards: Total Orders, Total Revenue, Cash payments (count + amount), Online payments (count + amount)
Pie Chart: Payment method distribution (Cash vs Online) using Recharts
Summary Statistics: Cash revenue, Online revenue, Average order value
Order History Table: Bill No, Date, Customer Name, Phone, Items, Discount, Total, Payment Method
CSV Export button to download filtered orders
Reset Data button (with confirmation) to clear all orders, reset bill counter to 1001, and clear customer records
Back button to return to POS

Data Model (localStorage)
Shop Info: name, address, contact, gstin, logo path
Categories: id, name, createdAt
Menu Items: id, name, price, categoryId, image, createdAt
Orders: id, billNumber, items[], subtotal, discountType (none/percentage/rupees), discountValue, discountAmount, cgst (0), sgst (0), total, paymentMethod, customerName, customerPhone, createdAt
Customers: id, name, phone, createdAt, lastOrderDate (auto-saved from orders)
Bill Counter: starts at 1001, auto-increments
Data Version: for cache-busting when menu data changes
